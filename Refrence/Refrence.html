Standard C++ Library reference

C Library

The elements of the C language library are also included as a subset of the C++ Standard library. These cover many aspects, from general utility functions and macros to input/output functions and dynamic memory management functions:

Library Name Category Description
<iostream> Standard I/O Provides basic input and output services for C++ programs.
<vector> Containers Provides a dynamic array implementation.
<string> Strings Provides string manipulation and operations.
<algorithm> Algorithms Provides a collection of functions for performing operations on ranges of elements.
<map> Containers Provides an associative container that stores key-value pairs.
<thread> Concurrency Provides facilities for creating and managing threads.
<fstream> File I/O Provides file stream classes for reading from and writing to files.
<cmath> Mathematics Provides common mathematical functions.
<memory> Memory Management Provides utilities for dynamic memory management, including smart pointers.
<chrono> Time Provides time-related utilities.

Containers

Container Header Description
std::vector <vector> A dynamic array that can resize itself automatically when elements are added or removed.
std::list <list> A doubly-linked list that allows fast insertion and deletion of elements anywhere in the sequence.
std::deque <deque> A double-ended queue that supports fast insertion and deletion at both the beginning and the end.
std::array <array> A fixed-size array container that provides a safer and more convenient interface compared to raw arrays.
std::set <set> A sorted associative container that contains unique elements.
std::map <map> A sorted associative container that stores key-value pairs with unique keys.
std::unordered_set <unordered_set> An unordered associative container that contains unique elements.
std::unordered_map <unordered_map> An unordered associative container that stores key-value pairs with unique keys.
std::stack <stack> A container adapter that provides a LIFO (Last-In-First-Out) data structure.
std::queue <queue> A container adapter that provides a FIFO (First-In-First-Out) data structure.
std::priority_queue <queue> A container adapter that provides a priority queue, where elements are ordered by priority.

Atomics and threading library

Component Header Description
std::atomic <atomic> Provides atomic operations on variables, ensuring thread-safe access and modification.
std::thread <thread> Represents a single thread of execution and provides mechanisms to create and manage threads.
std::mutex <mutex> Provides a mutual exclusion mechanism to protect shared data from simultaneous access by multiple threads.
std::lock_guard <mutex> A RAII-style wrapper for mutexes that automatically locks and unlocks the mutex.
std::unique_lock <mutex> Provides more flexible locking mechanisms compared to std::lock_guard, allowing deferred locking and manual unlocking.
std::condition_variable <condition_variable> Allows threads to wait for a condition to be met before proceeding.
std::future <future> Represents a future value that will be available asynchronously.
std::promise <future> Provides a mechanism to store a value that can be retrieved asynchronously via a std::future.
std::async <future> Runs a function asynchronously and returns a std::future that will hold the result.
std::atomic_flag <atomic> A simple atomic boolean flag that can be used for lock-free synchronization.
std::recursive_mutex <mutex> A mutex that can be locked multiple times by the same thread without causing a deadlock.
std::timed_mutex <mutex> A mutex that supports timed locking attempts.

Miscellaneous headers

Header Description
<utility> Provides utilities like std::pair and std::move for working with pairs and moving objects.
<algorithm> Provides a collection of functions for performing operations on ranges of elements, such as sorting, searching, and modifying sequences.
<chrono> Provides facilities for working with time points, durations, and clocks.
<functional> Provides function objects, binders, and other utilities for working with functions and callable objects.
<tuple> Provides the std::tuple class for working with fixed-size collections of heterogeneous values.
<optional> Provides the std::optional class for representing optional values that may or may not be present.
<variant> Provides the std::variant class for representing type-safe unions.
<any> Provides the std::any class for type-safe storage of single values of any type.
<bitset> Provides the std::bitset class for working with fixed-size sequences of bits.
<ratio> Provides compile-time rational arithmetic support.
<type_traits> Provides utilities for type introspection and type transformations at compile time.
<random> Provides facilities for generating random numbers using various distributions and engines.
<initializer_list> Provides the std::initializer_list class for working with brace-enclosed initializer lists.
